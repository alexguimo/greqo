Resumen de Generación y Archivos de Greqo

A continuación se presenta un recuento del desarrollo efectuado para el prototipo de Greqo, incluyendo todos los requerimientos especificados sobre diseño y funcionalidad de voz.

Lista de Archivos Generados en scratch/greqo

  • index.html
    Contiene la base semántica y estructurada del esqueleto web de la aplicación. Posee un visor/chat y un contenedor de controles muy minimalista en preparación al diseño gráfico. El botón principal está diseñado con su id correspondiente para ser enlazado por JS.
  • style.css
    Implementa la interfaz visual estilo *Inteligencia Artificial* tipo sistema IA clásico.
    • El fondo es oscuro y texturizado usando radial-gradient para dar profundidad.
    • Utiliza Glassmorphism mediante backdrop-filter: blur() y transparencias que simulan paneles de vidrio.
    • Diferenciación de Estados de Voz: Incorpora clases clave que producen animaciones de sombras (Box-Shadow) usando el sistema de *keyframes*.
      → Cian Neón (#00f0ff) para mostrar que Greqo está reconociendo activamente tu voz (Efecto expansión repetida corta).
      → Azul Neón (#0066ff) para cuando Greqo responde hablando (Efecto expansión rítmica e interna más densa).
  • script.js
    El motor funcional de este prototipo:
    • Hace uso de las APIs SpeechRecognition y speechSynthesis del navegador nativo con configuración es-CO (Español).
    • Dispone del token Bearer como constante en código crudo para poder hacer el fetch sin exponerlo en una interfaz gráfica.
    • Simulación Temporal: Incorpora dentro de enviarComando() un retardo (delay cronometrado en un setTimeout de 1.8 segundos) que hace al usuario percibir que transcurrió tiempo entre su consulta, el post de red y la deliberación de Greqo. Si la llamada POST falla, activa un motor interno que "responde temporalmente" permitiendo apreciar todo el loop de voz y UI.
  • lista.html
    Este archivo. Actúa como acta de entrega documentando qué fue editado y creado para finalizar la ejecución.
¡Todo el prototipo fue creado exitosamente usando solo HTML5, CSS y JavaScript Vanilla!
